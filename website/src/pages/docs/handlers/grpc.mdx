This Handler allows you to load [gRPC](https://grpc.io) definition files (`.proto`).

<PackageCmd packages={["@graphql-mesh/grpc"]} />

Now, you can use it directly in your Mesh config file:

```yaml {6}
sources:
  - name: MyGrpcApi
    handler:
      grpc:
        endpoint: localhost:50051
        protoFilePath: grpc/proto/Example.proto
```

## Use Reflection instead of proto files

If you have configured reflection in your gRPC server, you can use `useReflection` flag to load the schema from reflection instead of loading proto files.

```yaml {6}
sources:
  - name: gRPC Example
    handler:
      grpc:
        endpoint: localhost:50051
        useReflection: true
```

<Callout>
  You can check out our example that uses gRPC Handler with reflection.
  [Click here to open the example on GitHub](https://github.com/Urigo/graphql-mesh/tree/master/examples/grpc-reflection-example)
</Callout>

## Custom Metadata for Authorization

Here you can use `metaData` field to pass some custom metadata from the context.

```yaml {7-9}
sources:
  - name: MyGrpcApi
    handler:
      grpc:
        endpoint: localhost:50051
        protoFilePath: grpc/proto/Example.proto
        metaData:
          authorization: '{req.headers.authorization}'
          someStaticValue: 'MyStaticValue'
```
